## ASE Folder {#ase-folder}

### 1. `ASE Threshold data processing ultra.py`

**Purpose**: Main script for processing ASE threshold data. **Usage**: Enter the following in the GUI: - Calibration equation - Beam size - ND filter value used on the PD during measurement

**Key parameters**: - `Collapse factor`: controls the switching logic for the Savitzky-Golay smoothing window. When the current FWHM drops below **60%** of the historical maximum FWHM, the smoothing window is switched automatically. - `Smoothing window range`: configurable in the GUI. - `Auto_peak_window`: integration wavelength range after automatic peak detection. The two values below are only defaults; the program adjusts them automatically at each run, so **manual modification is not recommended**.

**Special algorithm**: Includes the **wing match** algorithm for removing the PL background. **Output**: Processed data file, including the `auto fit` field generated by `auto_threshold_module`.

------------------------------------------------------------------------

### 2. `auto_threshold_module.py`

**Purpose**: Submodule responsible for automatic fitting and ASE threshold extraction. **Description**: **Cannot run independently**; it is called internally by `ASE Threshold data processing ultra.py`. **Note**: It must be placed in the same folder as `ASE Threshold data processing ultra.py`. **Output**: In the final saved data file, the `auto fit` column is generated by this module.

------------------------------------------------------------------------

### 3. `Edge loss data process.py`

**Purpose**: Processes edge-loss data. **Output**: The generated final file can be used directly for plotting and loss calculation.

------------------------------------------------------------------------

### 4. `Threshold data processing steps.py`

**Purpose**: A **step-by-step demonstration version** showing how the wing match algorithm removes the PL background. **Usage**: Select any folder and run the script. It generates four intermediate images showing the spectrum at each processing stage.

------------------------------------------------------------------------

### 5. `ASE_manual_fit.py`

**Purpose**: **Manual piecewise linear fitting GUI** for ASE / DFB threshold data. It reads the incident energy vs integrated intensity curve from `*_Analysed_ultra.xlsx`, allows interactive removal of outliers, and determines the threshold turning point. **Usage** with the CustomTkinter interface:

1.  After launch, click "Select Folder (Analysed_ultra)" and choose a folder containing exactly one `*_Analysed_ultra.xlsx` file.
2.  Select 2 / 3 / 4 linear segments, corresponding to 1-3 turning points.
3.  **Left click** a data point to exclude it from both fitting and plotting; **right click** to exclude it from fitting only while keeping it in the plot.
4.  View turning-point coordinates and errors, segment slopes, slope ratios, and R² in real time, then click "SAVE RESULTS" to confirm.

**Output**: - `ManualFit_Result_{base}.xlsx`, containing the Parameters / Data_and_Mask / Plot_data / Fit_Line worksheets. - `{base}_ManualFit_Plot.png`. - If a same-name `{base}_auto_fit.xlsx` file exists, it is removed because manual results take priority. **Description**: Runs independently and has no sibling-module dependency.

------------------------------------------------------------------------

### 6. `Abs_percentage_TMM.py`

**Purpose**: Calculates thin-film absorption $A = 1 - R - T$ using the **transfer-matrix method (TMM)**. It reads $n$ and $k$ data exported from CompleteEASE and calculates normal-incidence $R$, $T$, and $A$ as functions of wavelength for a given film thickness. **Model**: Three-layer structure: air ($n=1.0$) / film / glass substrate ($n=1.52$). It uses `tmm.unpolarized_RT` wavelength by wavelength and annotates the absorption at 330 nm and the absorption peak above 400 nm.

**Usage**: - Run without arguments to open file selection and film-thickness input dialogs. - Command line: `python Abs_percentage_TMM.py "path/to/nk data.txt" 120`, where the second argument is film thickness in nm.

**Output** in the same directory as the nk file: - `{stem}_absorption_TMM.csv` with wavelength_nm, R, T, and A_percent. - `{stem}_absorption_TMM.pdf` at 600 dpi. **Dependencies**: `tmm`, `pandas`, `numpy`, `matplotlib`; the last path is remembered in `abs_tmm_last_path.json`.

------------------------------------------------------------------------

### 7. `Oceanoptics data process.py`

**Purpose**: **Ocean Optics spectrum CSV batch-processing tool**. It recursively scans folders, removes PL background from ASE spectra, calculates FWHM and integrated intensity, and generates summary plots and Excel files. The algorithm is ported from `ASE Threshold data processing ultra.py`, but this script is standalone. **Key logic**: - Wavelength range is cropped to 390-800 nm, configurable near the top of the script. - Files whose names contain `background` are excluded. - The peak is detected automatically, and the integration window is peak ± 10 nm. - The PL reference is built from the average of the three weakest spectra, smoothed with Savitzky-Golay, scaled, and subtracted outside the integration window. - FWHM is taken as the median result from multiple Savitzky-Golay smoothing windows.

**Usage**: Run the script and select the root directory; it recursively processes subfolders containing CSV files. **Output** in each processed folder: - `Results_{folder}.xlsx`. - `Summary_Intensity_FWHM_{folder}.png` with dual Y axes. - `All_Normalized_Spectra_{folder}.png`.

## DFB devices Folder {#dfb-devices-folder}

### 1. `auto_threshold_module_laser.py`

**Purpose**: Automatic threshold fitting module for DFB laser data, implemented as a variant of `auto_threshold_module`. **Background**: Because the laser source used for SA testing is extremely unstable, raw data can contain many outliers. **Features**: Includes an algorithm for automatically rejecting unreasonable points, although it is not always effective. **Description**: **Cannot run independently**; it must be called by a main script.

------------------------------------------------------------------------

### 2. `Bilinear_fit.py`

**Purpose**: Manual fitting script. **Usage**: 1. Run the script and select the ultra data file generated by `ASE Threshold data processing ultra`. 2. In the popped-up UI plot, click data points to control whether each point participates in fitting. 3. Observe threshold and other parameter changes in real time. 4. Click "Save" to output: - A fitting preview image in PNG format - An xlsx data file that can be used for plotting

------------------------------------------------------------------------

### 3. `Laser Threshold data processing.py`

**Purpose**: ASE main-script variant for DFB data. **Differences from the ASE version**: - **No** PL background removal algorithm. - The saved data file contains only **integrated intensity** and **peak-optimized FWHM results**.

------------------------------------------------------------------------

### 4. `DFB_FWHM_math_core.py`

**Purpose**: **Deterministic calculation library** for organic DFB laser spectrum FWHM, intended to be imported by other scripts. It specifically addresses premature FWHM collapse caused by noise spikes before threshold. **Algorithm**: - Preprocessing: 3-point median filter plus a wide Savitzky-Golay envelope. - Narrow coherent components are extracted from the residual signal, defined as cleaned signal minus envelope. If both the topological criterion, residual peak prominence >= 10% of the envelope peak, and the energy criterion, residual energy fraction >= 15%, are satisfied, the spectrum is classified as "lasing" and FWHM is measured on the cleaned signal; otherwise FWHM is measured on the envelope, representing broad PL/ASE. - Sub-pixel half-maximum width: half-maximum crossings are obtained by linear interpolation. **API**: `calculate_dfb_fwhm_adaptive(wavelength, intensity, ...)`, with optional `return_diagnostics=True`. **Description**: Pure module, **cannot run independently**, and depends only on numpy and scipy.

------------------------------------------------------------------------

### 5. `Get_pixel_area.py`

**Purpose**: Analyzes two-dimensional beam intensity matrices exported from **BeamView** and calculates beam area using the **1/e² criterion, 13.5% of peak intensity**. **Logic**: Reads `PixelWidth` / `PixelHeight` in µm from the CSV header, counts pixels above threshold, and converts the result to physical area in µm² / mm² / cm². **Usage**: Run the script and select a BeamView-exported `.csv` or `.txt` file. **Output** in the same directory: - `{base}_profile_clean.svg` without contour lines. - `{base}_profile.svg` / `.pdf` with a red 1/e² contour line at 600 dpi. - Pixel count and physical area are printed to the console.

------------------------------------------------------------------------

### 6. `SE_cross_section.py`

**Purpose**: **Stimulated emission (SE) cross-section calculator GUI**. It calculates $\sigma_{em}(\lambda)$ from a PL spectrum, refractive index $n$, and radiative lifetime using the Füchtbauer-Ladenburg relation, following Nakanotani et al., *Adv. Opt. Mater.* 2017. **Formula**: $\sigma_{em} = \lambda^4 E_f / (8\pi n^2 c\,\tau_{rad})$ in CGS units, with the result in cm², where $\tau_{rad} = \tau_f / \mathrm{PLQY}$ and $E_f$ is obtained by area-normalizing the PL spectrum. **Usage** with CustomTkinter: import PL data and $n$ data, enter $\tau_f$ in ns and PLQY, then click CALCULATE -> SAVE RESULTS. **Output**: CSV with Wavelength_nm, Ef_per_cm, n, sigma_em_cm2, including a parameter header; default name is `{PL_stem}_SE_cross_section.csv`.

------------------------------------------------------------------------

### 7. `Spectrum wavelength offset correction.py`

**Purpose**: Batch **correction of wavelength-axis offsets** in spectrum CSV files. It recursively scans folders and applies a fixed offset in nm **in place** to CSV files whose names contain `spec`. **Configuration items** near the top of the script: `WAVELENGTH_OFFSET`, default -1 nm; `TARGET_FORMAT`, one of `both`, `standard`, or `transpose`; and `BASE_FOLDER`. **Usage**: Configure the constants and run the script. If `BASE_FOLDER` is empty, a folder-selection dialog opens. Files whose names contain `extract` or `process` are excluded. **Output**: Matching spec CSV files are **overwritten in place** with no new output file; the console prints the format and corrected point count for each file. **Note**: This directly rewrites raw data, so back up files first.

------------------------------------------------------------------------

## KIT Transient Absorption Folder {#kit-transient-absorption-folder}

### 1. `mat_to_CSV.py`

**Purpose**: Batch converts transient absorption (TA) `.mat` files exported from KIT / LabVIEW into CSV files, making them easier to use in Excel or `TA_data_reading.py`. **Features**: - Reads the `TA_data` structure: `Stage_positions` for time, `wavelength_calibration` for wavelength, and `dT_T` for the signal matrix. - Supports MAT v7.3 through `mat73`, standard v5 through `scipy`, and non-standard v5 through `mat4py`. - Detects Google Drive / OneDrive placeholder files and prompts the user to download them locally first. - Uses `ProcessPoolExecutor` for folder-level parallel batch conversion. **Usage**: Run the script, select a folder, and recursively scan all `.mat` files. **Output**: A same-name `.csv` file is generated next to each `.mat`; rows are wavelength in nm and columns are time in s.

------------------------------------------------------------------------

### 2. `TA_data_reading.py`

**Purpose**: **Interactive visualization tool** for KIT TA data. It loads a two-dimensional matrix, wavelength × time, from `.xlsx` files and supports extracting spectra at fixed times or kinetic traces at fixed wavelengths. **Core capabilities**: - Two modes: `Time -> spectrum` for fixed-time spectra and `Wavelength -> trace` for fixed-wavelength traces. - Optional moving averages along time / wavelength axes: 10 ps / 1 ns / 2 ns; 2 nm / 3 nm. - Multiple curve overlay, nearest-point readout on mouse hover, Matplotlib toolbar zooming, and Windows high-DPI adaptation. **Usage**: Browse / Load an `.xlsx` file, configure parameters, click **Add Curve**, and use **Clear Curves** to reset. **Output**: No file output; interactive plot only.

## TRPL/PL Subfolder {#trplpl-subfolder}

### 1. `Voigt_PL.py`

**Purpose**: Performs **Voigt profile fitting** for steady-state PL emission spectra. It currently supports two analysis modes, selected manually at startup or specified with `--mode`:

| Mode | `--mode` | Description |
|--------------------|--------------------------------|--------------------|
| Gamma sensitivity scan | `sensitivity` | Folder-wide double-Voigt fitting, 0-0 + 0-1, scanning $\sigma$ under fixed $\gamma$ |
| RT + 77K global joint fitting | `global` | Manually select RT and 77K spectra; shared $\sigma$, independent $\gamma$ |

> **Description**: The older standalone `77k-dual` mode has been removed from the `--mode` options. Its 77K double-peak logic is now integrated into `global` mode. `--global-fit` is deprecated and is equivalent to `--mode global`.

**Data preprocessing**, shared by both modes: `load_pl_csv` reads `.csv` spectra, with the first column as wavelength in nm and the second as intensity, including support for Jasco headers. Wavelength is converted to energy by $E = 1239.8 / \lambda$, and intensity is Jacobian-corrected as $I_E = I_\lambda \cdot \lambda^2$. Peak position and FWHM are estimated automatically and fitting is performed in a local window. CSV files whose names contain `Gated` are skipped automatically. The baseline is fixed at 0.

**Usage**:

``` bash
# Folder-selection dialog + mode selection
python Voigt_PL.py "data folder"

# Mode 1: gamma sensitivity scan, whole folder
python Voigt_PL.py "data folder" --mode sensitivity
python Voigt_PL.py "data folder" --mode sensitivity --workers 4

# Mode 2: RT + 77K global joint fitting
python Voigt_PL.py "data folder" --mode global
python Voigt_PL.py "data folder" --mode global --rt-csv "xxx_RT.csv" --77k-csv "xxx_77K.csv"
```

**Dependencies**: `numpy`, `pandas`, `matplotlib`, `lmfit`, `scipy`, `tqdm`; GUI operation requires `tkinter`. Plotting and summary export are provided by the sibling module `Voigt_PL_output.py`, described below.

------------------------------------------------------------------------

#### Mode 1: `sensitivity`, Gamma Sensitivity Scan

**Fitting logic**: Double Voigt, 0-0 + 0-1, with 0-1 fixed about 0.15 eV below 0-0. For each spectrum, fitting is performed with fixed $\gamma$ over $\gamma = 10\text{--}30\ \mathrm{meV}$ in 2 meV steps, recording $\sigma$, center, $R^2$, and related parameters. `ProcessPoolExecutor` provides parallel execution, and `tqdm` displays progress.

**Output** in `Voigt Output/`:

- `voigt_gamma_sensitivity_summary.xlsx`: one row per `(filename, gamma)` in **scan-table format**.
- `{sample_name}_gamma20meV_fit.png`: fit plot at $\gamma = 20\ \mathrm{meV}$, including Voigt plus Gaussian/Lorentzian decomposition.
- `sigma_vs_gamma_sensitivity.png` / `.svg`: $\sigma$ vs $\gamma$ sensitivity line plot.

------------------------------------------------------------------------

#### Mode 2: `global`, RT + 77K Global Joint Fitting

**Model**:

- **RT**: single Voigt.
- **77K**: double Voigt, 0-0 + 0-1, with `center_01 = center_00 - E_\mathrm{vib}` and $E_\mathrm{vib}$ constrained to **0.15-0.18 eV**, typical of C-C backbone stretching.
- **Shared** `sigma_global` across RT and both 77K peaks; `gamma_RT` and `gamma_77K` are independent, with `gamma_77K < gamma_RT`.

**Output** in `Voigt Output/`:

- `global_voigt_fit_summary.csv`: global parameter summary.
- `global_RT_77K_voigt_fit.png`: two panels, RT single peak and 77K double peak with dashed component curves.

------------------------------------------------------------------------

#### Plot Style

- Figure size 20 cm × 15 cm, 16 pt font; ticks only on the left and bottom axes, with no top/right ticks.
- The sensitivity-line legend is placed outside the plotting area on the right; `savefig(..., bbox_inches='tight')` prevents clipping.

------------------------------------------------------------------------

### 2. `Voigt_PL_output.py`

**Purpose**: **Plotting and parameter-export utility library** for `Voigt_PL.py`; it is not a main program and **cannot run independently**. It provides journal-style plotting and writes fitting summaries to CSV / XLSX files. **Core functions**: - `apply_plot_style`: 20 × 15 cm, 16 pt font, no top/right ticks, no grid. - `plot_representative_fit_77k_dual` / `plot_representative_fit`: 77K 0-0 / 0-1 double-Voigt components plus total fit. - `plot_global_fit_comparison`: RT and 77K global joint fitting comparison plot. - `plot_sigma_sensitivity`: $\sigma$ sensitivity line plot under fixed $\gamma$. - `save_global_fit_summary` / `save_77k_dual_fit_summary` / `save_sensitivity_summary_xlsx`: parameter and summary-table export. **Description**: Imported by `Voigt_PL.py` and used through injected callbacks; output defaults to `Voigt Output/`, created by `ensure_output_dir`.

------------------------------------------------------------------------

### 3. `Remove_Peak.py`

**Purpose**: Processing script for **136 ps gated PL at 80 K**. It subtracts prompt fluorescence (PF) using an **RT PL template** to extract a clean phosphorescence spectrum. **Algorithm**: - Automatically identifies the first two numeric columns in CSV files, supporting multiple encodings and delimiters; optional dark subtraction and 420-450 nm baseline correction. - Converts between wavelength and energy domains using the Jacobian transformation with 1240 eV·nm. - Geometric edge matching: adjusts RT-template shift in eV and gamma broadening to match the half-maximum and 1/e crossing points on the blue edge of 80 K PF; solves the scale factor $\alpha$ by least squares in the PF blue-edge window, 2.620-2.780 eV. - Cosine taper weighting: subtraction gradually stops below the PF peak to avoid damaging the phosphorescence region; negative values are clipped to 0.

**Usage**: - Command line: `python Remove_Peak.py [80K.csv] [RT.csv] [--dark-80k ...] [--dark-rt ...] [--manual-alpha/shift/gamma ...]`, etc. - If paths are omitted, dialogs ask for the 80 K and RT spectra in sequence. **Output** based on the 80K filename: - `*_phosphorescence_fit_table.csv`. - `*_phosphorescence_fit_plot.png` in energy domain and `*_phosphorescence_fit_plot_lambda.png` in wavelength domain. - `*_phosphorescence_fit_params.json`, with optional `*_clean_phosphorescence_lambda.csv`.

### 4. `PL_onset_runner.py`, PL Onset Detection

**Path**: `TRPL/PL/`, with sibling modules `onset_calculator.py`, `plotter.py`, and `filters.py`.

**Purpose**: Automatically detects **onset wavelength / energy** from steady-state PL emission spectra exported as CSV/TXT by Jasco and similar instruments, and generates validation plots with baseline, tangent, and onset markers. Supports both single-spectrum analysis and fluorescence / phosphorescence two-spectrum comparison.

**Algorithm, Tangent-Baseline Intersection in the energy domain**:

1.  Reads numeric rows from the first two CSV columns, defaulting to wavelength in nm and intensity in counts, while ignoring file-header metadata.
2.  Converts $I(\lambda)$ to photon-energy coordinates and applies the Jacobian correction: $$I(E) = I(\lambda)\,\frac{\lambda^2}{hc},\qquad hc = 1240\ \mathrm{eV{\cdot}nm}$$
3.  Smooths $I(E)$ in the **energy domain** using Savitzky-Golay filtering, takes the tangent at the maximum derivative, and intersects it with the baseline to obtain the onset.
4.  **Baseline**: average intensity of the highest-energy **10%** of data points after sorting; configurable through `BASELINE_REGION` in `PL_onset_runner.py`.
5.  **Tangent search region**: from the main peak toward the high-energy side, `post_peak`, avoiding accidental selection of the falling edge.
6.  Results are reported in both nm and eV; dual-spectrum mode additionally reports $\Delta\lambda$, flu - phos, and $\Delta E$.

**Usage**, run from `TRPL/PL/`:

``` bash
# Dialog selection for single / dual spectrum mode
python PL_onset_runner.py

# Single spectrum, CSV specified on command line
python PL_onset_runner.py "spectrum.csv"

# Fluorescence + phosphorescence comparison, two CSV files
python PL_onset_runner.py --dual "fluorescence.csv" "phosphorescence.csv"
```

**Output** in the same directory as the input CSV; vector graphics are **SVG** with 600 dpi rendering settings:

| Mode | Wavelength-coordinate plot | Energy-coordinate plot |
|------------------|---------------------------|---------------------------|
| Single spectrum | `{stem}_onset.svg` | `{stem}_onset_energy.svg` |
| Dual spectrum | `{flu}_vs_{phos}_onset_diff.svg` | `{flu}_vs_{phos}_onset_diff_energy.svg` |

- **Wavelength plot**: X axis in nm with a top axis in eV; interactive preview opens when `show_plot=True`.
- **Energy plot**: X axis in eV with a top axis in nm; same content as the wavelength plot, including spectra, baseline, tangent, and onset, saved in the background without an extra popup.
- **Dual-spectrum plot**: two panels: raw counts and individually normalized spectra from 0 to 1; the summary box annotates both onsets plus $\Delta\lambda$ and $\Delta E$.

**Default parameters** near the top of `PL_onset_runner.py`: `BASELINE_REGION = 0.10`, `WINDOW_LENGTH = 11`, `POLYORDER = 3`, `SAVE_DPI = 600`.

**Figure style**: 20 cm × 15 cm, Arial 20 pt, inward ticks, and Y-axis scientific notation included in the axis title.

**Related modules**: `onset_calculator.py` for file reading and energy-domain onset calculation, `plotter.py` for wavelength / energy validation plots and dual-spectrum comparison plots, and `filters.py` for Savitzky-Golay filtering. The last data directory is recorded in `pl_onset_last_path.json`.

**Dependencies**: `numpy`, `matplotlib`; GUI operation requires `tkinter`.

## TRPL/TCSPC Subfolder {#trpltcspc-subfolder}

> This subdirectory is the **time-correlated single-photon counting (TCSPC) lifetime fitting stack**. The unified main program is `mainUI_lifetime_fit.py`; the other files are engines and utility modules orchestrated by it.

### 1. `mainUI_lifetime_fit.py`, Main GUI Program

**Purpose**: **Unified interactive GUI for TRPL lifetime fitting**, integrating Reconvolution Fit and Tail Fit. It is the evolved / renamed version of the older `lifetime_fit.py`. **Core capabilities**: - **Dual-mode switching**: choose reconvolution fitting or tail fitting at startup. - **Intelligent IRF management**: in reconvolution mode, automatically searches for and matches IRF CSV files named with `IRF`, then aligns the rising edge. - **Integrated feature set**: parameter configuration, xmin/xmax range adjustment, up to 4 $\tau/\beta$ components with optional fixing, phosphorescence marking, Area Diff for PF/DF, extrapolation / Scatter options, PLQY and PF/DF component selection, **kinetic rate analysis (RISC)** switch, and JSON configuration save/load. - **Export consistency**: stores both the UI-displayed $\tau_{eff}$ and the kinetics-specific $\tau_{mean}$.

**Usage**: `python mainUI_lifetime_fit.py` or `python mainUI_lifetime_fit.py path/to/decay.csv` -> choose mode -> select CSV, plus IRF for Recon -> Open Fit Menu -> Run Fit -> Save Results. **Output**: User-selected `*.xlsx` file with Parameters and Fit_Curve worksheets, plus a same-path `*.png` at 400 dpi; runtime paths are written to `last_paths.json`.

**Call-chain overview**:

```         
mainUI_lifetime_fit.py
  ├─ Recon_fit_process / Tail_fit_process      (fitting engines)
  │    ├─ fit_multistart        (parallel multi-start least_squares)
  │    ├─ fit_uncertainty       (JᵀJ covariance + delta-method errors)
  │    ├─ Area_Analysis_Engine  (PF/DF area-difference method)
  │    ├─ params_export         (Excel parameter-row assembly)
  │    └─ risc_calculator_bridge(optional RISC calculation + panel text)
  └─ risc_calculator_bridge      (left-side parameter summary panel layout)
```

------------------------------------------------------------------------

### 2. `Recon_fit_process.py`

**Purpose**: Core mathematical engine for reconvolution fitting in `mainUI_lifetime_fit.py`, responsible for low-level physical calculation. **Core capabilities**: - **Reconvolution algorithm**: uses FFT convolution to combine the instrument response function (IRF) with the decay model, then fits TCSPC data accurately by least squares. - **Stretched-exponential physical model**: fits multi-component stretched exponentials and automatically calculates mean lifetime ($\langle \tau \rangle$). - **Weighted average lifetimes**: intensity-weighted $\tau_{eff}$ for UI display and number-weighted $\tau_{mean}$ for kinetic-rate input, exported as `Hidden_Number_Mean_Avg_Tau`. - **PF/DF quantification**: built-in area-difference method for separating PF / DF / phosphorescence fractions, calling `Area_Analysis_Engine`. **Description**: **Cannot run independently**; called by the main program.

------------------------------------------------------------------------

### 3. `Tail_fit_process.py`

**Purpose**: Core mathematical engine for tail fitting in `mainUI_lifetime_fit.py`. **Core capabilities**: - **Pure exponential / stretched-exponential fitting**: fits multi-component decay tails without IRF convolution. - **Aligned logic**: follows the reconvolution version for mean-lifetime analysis, with $\tau_{eff}$ used in the UI and $\tau_{mean}$ used for kinetics, as well as PF/DF ratio calculation. **Description**: **Cannot run independently**; called by the main program.

------------------------------------------------------------------------

### 4. `Area_Analysis_Engine.py`

**Purpose**: Independent physical module for calculating **PF / DF areas** from stretched-exponential fitting components, shared by both Recon and Tail routes. **Algorithm**: - `calc_recon_difference_areas`: Recon area-difference method, `PF = raw integral - DF - Phos [- Scatter]`; no extrapolation is needed because the fitting window already includes the IRF rising edge. - `calc_tail_difference_areas`: Tail area-difference method, with optional geometric extrapolation compensation. It uses the pure-shape area ratio of C1, $\tau_1/\beta_1$, to correct the missing PF before xmin. **Description**: Pure calculation module that returns dictionaries and writes no files; it is called by the engines when "Enable Area Diff (PF/DF Ratio)" is selected.

------------------------------------------------------------------------

### 5. `fit_multistart.py`

**Purpose**: **Parallel multi-start optimization wrapper** for lifetime fitting. Without changing the residual or model mathematics, it runs `least_squares` from multiple $\tau/\beta$ initial guesses and selects the result with the lowest reduced $\chi^2$. **Features**: Performs logarithmic scans of $\tau$ and $\beta$ seeds around the user's initial values, with the exact user-provided initial values always used as start #0. Supports up to 8 starts and a persistent `ProcessPoolExecutor`, friendly to Windows spawn behavior. **Description**: **Cannot run independently**; depends on `Recon_fit_process` / `Tail_fit_process` and is called by their `run_fitting_process`.

------------------------------------------------------------------------

### 6. `fit_uncertainty.py`

**Purpose**: Estimates parameter uncertainties from **Poisson-weighted residuals** and propagates errors to derived lifetimes using the **delta method**. **Algorithm**: TCSPC counts use Poisson weights $1/\sqrt{\max(N,1)}$. At the optimum, the Jacobian estimates $\mathrm{Cov}(p) \approx (J^\top J)^{-1}$; errors are then propagated to intensity- and number-averaged $\tau$. Provides `format_val_err` / `format_beta_err` for UI display. **Description**: Library module called by the Recon / Tail engines after optimization.

------------------------------------------------------------------------

### 7. `params_export.py`

**Purpose**: Builds row lists for export to the Excel `Parameters` worksheet, especially the input / output documentation blocks for the **RISC Calculator**. **Core functions**: `append_section` for blank-line separation between modules, `build_risc_excel_input_rows` for inputs such as $\tau_p$, $\tau_d$, $\Phi_{PF}$, and $\Phi_{PLQY}$, and `build_risc_approx_output_rows` for Masui / Dias / Wada approximate $k_{RISC}$. **Description**: Library module that returns in-memory rows and does not write files by itself.

------------------------------------------------------------------------

### 8. `risc_calculator_bridge.py`

**Purpose**: Uses **xlwings** to drive the sibling `RISC Calculator.xlsx` file, opened read-only and not saved, to calculate exact / approximate TADF RISC rates. It also provides the parameter summary panel layout in the main GUI left column. **Core capabilities**: - Writes D5, $\tau_p$ in ns; D8, $\tau_d$ in µs; D11, $\Phi_{PF}$; and D13, $\Phi_{PLQY}$; iteratively solves for $k_{ISC}$, then reads exact and approximate solutions. - `compute_risc_rates` / `compute_risc_rates_batch` for Monte Carlo batch calculation. - `create_lifetime_fit_figure` / `FitSummaryPanel` for the colored left-column parameter summary. **Usage**: Automatically called when "Compute RISC Rates (Excel)" is checked in the main workflow; can also run its built-in example with `python risc_calculator_bridge.py`. **Hard dependencies**: Sibling `RISC Calculator.xlsx`, `xlwings`, and local Microsoft Excel.

------------------------------------------------------------------------

### 9. `RISC_exact_solution_coremath.py`

**Purpose**: **Pure-Python exact analytical solution engine** for TADF kinetic rates, based on Cardano's formula for cubic equations. It coexists as a parallel / verification path outside the Excel bridge. **Core functions**: - Extracts $k_{RISC}$, $k_{ISC}$, $k_r$, $k_{nr}$, and related quantities from experimental data; it is more accurate than approximate formulas when the delayed component fraction is high. - Strictly requires **number-weighted lifetime** input, $\tau_{mean}$, to match the physical definition of rate constants. - Automatically corrects the DF component and calculates $\Phi_{PF}$, $\Phi_{DE}$, $\Phi_{DF}$, and related quantities. **Description**: The current main workflow primarily uses `risc_calculator_bridge.py`, the Excel route; this module remains available as an independent exact-solution implementation for import or validation runs. The old standalone GUI `RISC_exact_solution.py` has been removed.

## Python plot Folder {#python-plot-folder}

Scripts related to **local interactive plotting** of spectra, thresholds, decays, and TRPL batch preprocessing are located in the **`Python plot/`** subdirectory.

### Manifest and Multi-Curve Plotting, Join Curves

- **`Manifest_index.py`**: After a data root directory is selected in a dialog, recursively scans `*.csv`, `*.txt`, and `*.xlsx` files and generates **`plot_manifest.xlsx`**, with columns including `File_Path`, `Rel_Path`, and `Label`, providing a unified path registry for downstream plotting.
- **`Join_curves.py`**: Reads the manifest and launches a **Dash** web interface. Users can select multiple curves for comparison in one plot, with support for axis titles, colors, normalization, logarithmic Y axis, legend control, and vector export. Configuration is written to **`join_curves_config.json`** in the same directory, and confirmation exports **PDF/SVG** files. Run it from the **`Python plot/`** directory with `python Join_curves.py` so same-level modules such as `PlotUtils` import correctly.
- **`Join_curves_nonDash_core.py`**: **Dash-free core module**. Manifest parsing with `parse_manifest`, data reading, series collection with `collect_series`, Plotly preview creation, and Matplotlib static export are all implemented here. It is used by `Join_curves.py` and can also be imported independently for batch processing or further development.

### Shared Style and Other Python-Embedded Plotting

- **`PlotUtils.py`**: Matplotlib styling, `create_matched_fig_ax`, global font sizes, multi-curve color palettes through `SPECTRA_COLOR_PALETTES`, `DynamicPlotExplorer`, and other utilities shared by Join_curves, ASE_graph, and related scripts.
- **`ASE_graph.py`**: Plotly preview and Matplotlib export for ASE threshold data. The interaction mode is similar to Join_curves, and it succeeds / renames the older `Threshold_graph.py`. It adds an `Analysed_ultra` spectral-evolution plot on top of the threshold plot. Exports `Threshold_graph.pdf/.svg`; if spectral data exists, also exports `Spectral_Evolution.pdf/.svg`. Configuration is stored in `threshold_plot_config.json`. **Dependency**: sibling `PlotUtils.py`.
- **`Decay_graph.py`**: Plotting for time-resolved decay curves, including Time Scan logic.
- **`Laser_graph.py`**: Multi-file comparison and plotting for DFB laser data.

### TRPL Batch Preprocessing, Located in `Python plot/`

- **`Draw_mutiple_graphs.py`**, note the spelling *mutiple*: Advanced batch-processing script for large numbers of TRPL / PL / UV-Vis data files. It recursively scans all Jasco `.csv` / `.txt` files in a folder and processes them in parallel by subfolder. For TRPL, it aligns all curves to the latest rising edge in the folder and normalizes the baseline to 1. For PL emission scans, it detects peaks, calculates FWHM and integrated intensity, and writes them into the title. For UV-Vis, it annotates absorption at 330 nm and above 400 nm. Generates a same-name `.png` at 400 dpi next to each source file. **Dependency**: sibling `trpl_processor.py`.
- **`trpl_processor.py`**: Core TRPL alignment / normalization engine, including `align_baseline_to_one`. **Cannot run independently**; it is the mathematical backend for `Draw_mutiple_graphs.py`.

## Python with origin Folder {#python-with-origin-folder}

The following scripts are used to **write data into Origin worksheets and call Origin for plotting**. This workflow differs from the `Python plot/` route, which uses Dash / Matplotlib **inside Python**. Template files are located in `Python with origin/origin_template/`.

- `Decay_graph_origin.py`: Imports decay data into Origin and plots it with automatic time-unit handling.
- `Threshold_graph_origin.py`: Creates Origin plots for ASE threshold fitting results.
- `Laser_graph_origin.py`: Origin visualization script for DFB laser data.
- `Scan_decay.py`: Batch version of `Decay_graph_origin.py`, used to scan and plot decay curves in batches. **Currently not used; stability is unknown**.

## Root-Level Script {#root-level-script}

### `Save any pic.py`

**Purpose**: Generates a high-resolution PNG "folder note / warning" image from custom text entered through a GUI. It is useful for marking folder status in File Explorer, for example "Data invalid due to low signal". **Usage**: Enter the note text in the input box and choose a save location; the default output name is `IMPORTANT_NOTE.png`. **Output**: 600 DPI PNG with dark purple background, white centered text, and automatic line wrapping. **Dependencies**: `tkinter` + `Pillow`; runs independently.
